02 AI Processing Pipeline
STEP 01
Ingest & Parse
Upload PDF/DOCX/TXT → extracts raw text + structure → stored in S3 and Postgres
STEP 02
PII Scrub
Presidio removes name, photo references, address, graduation year before any LLM call
STEP 03
Structured Extract
Claude extracts skills, titles, tenure, education, and achievements as typed JSON — no hallucination on factual fields
STEP 04
Embed & Index
Skill + experience text embedded via text-embedding-3-small; upserted to Pinecone under job namespace
STEP 05
Score & Rank
Weighted scoring: skill match (40%) · experience fit (30%) · trajectory (20%) · culture signals (10%)
STEP 06
Explain & Audit
Claude generates plain-English rationale per candidate; adverse impact 4/5ths rule check logged per run
Design Principle

PII removal happens before any LLM call — the model never sees a name, address, or photo. Bias mitigation before any scoring occurs.

03 Feature Set
⚖️
Bias Audit Dashboard
Runs EEOC 4/5ths rule check on every screening batch. Flags adverse impact by inferred demographic proxies and surfaces it to admins before shortlists are shared.
💬
Explainable Scores
Every candidate score includes a plain-English explanation generated by Claude — "Strong fit: 8 years Python, led distributed systems team, OSS contributor" — not just a number.
📄
Bulk Resume Ingestion
Upload 1–1000 resumes via dashboard or POST to /api/v1/resumes/batch. Celery workers process asynchronously; recruiter notified via webhook when complete.
🎯
Custom Scoring Weights
Hiring managers adjust skill/experience/culture signal weights per role via a simple UI slider — a staff engineer role weights technical depth heavier than a sales role.
🏷️
Skills Taxonomy
Proprietary skills graph normalises "React.js", "ReactJS", and "React" to a single node. Edges encode skill adjacency — a Rails dev scores partial credit for Django experience.
🔐
Role-Based Access
Admins see all; hiring managers see their own roles; recruiters see shortlists only — no raw resume access unless explicitly granted. Full access log for GDPR compliance.
04 Key Architectural Decisions
Decision Choice Made Why
PII handling strategy Strip before LLM, store raw encrypted We strip before the model screens resumes. FairHire AI model never processes names. Raw resumes stored in database for audit; model-processed data is always anonymised.
Scoring architecture Weighted rubric (not end-to-end ML) Our model offers transparent rubric with configurable weights that is auditable and easily explainable.
05 Bias Mitigation Strategy
Structural
Pre-LLM PII Scrubbing
Name, gender pronouns, address, photo descriptions, and graduation year are removed from the text before any LLM or embedding call. This action is embeded in the structore and not negotiable.
Monitoring
EEOC 4/5ths Rule Check
After every shortlist generation, the system calculates selection rates across inferred demographic proxies.
Scoring Design
Skills-First Rubric
The scoring rubric is anchored on demonstrated skills and measurable achievements . Hiring manager can adjust the scoring matrix weights to put more weight on education or skills, but there is no weight on a type of university or what company employee previously worked for.
Audit Trail
Explainable Score Ledger
Every scoring event is written to an append-only audit logwhich provides defensible records if a hiring decision is ever legally challenged.
Human-in-Loop
AI Recommends and Humans Decide
FairHire AI produces a ranked shortlist with explanations — it never auto-rejects a candidate. Every action (meets or does not meet requirements) requires a human approval and allows for an override.
06 Core Data Model
Entity Key Fields Notes
Candidate id, resume_id, skills[], embedding_id, anonymised_profile Decoupled from Resume to support one person applying to multiple roles. Anonymised profile is the model's view — never contains PII.
AuditLog id, event_type, actor_id, target_id, payload_hash, timestamp Append-only table. Payload is hashed, not stored, to avoid PII in logs. Covers scoring runs, shortlist views, status changes, and exports.
